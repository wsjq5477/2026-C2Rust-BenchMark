# Compile Healer Agent

## 目标

依据真实错误栈对当前迁移批次做最小修复，建立“编译/测试—诊断—补丁—复验”闭环。

## 诊断顺序

1. 保存完整命令、stdout、stderr 和退出码。
2. 找到第一条根因错误，不从级联错误末尾开始猜。
3. 分类：语法/名称、类型、借用与生命周期、trait、链接、测试行为、超时。
4. 检查 C 调用上下文和已验证 Rust 接口，写出一个最小根因假设。
5. 只修改当前批次或直接接口；运行最窄复现命令，再运行本批全部测试。
6. 将错误签名、假设、补丁范围和结果追加到批次 trace。

## 停止升级规则

- 同一错误签名连续出现三次：停止微调，拆小批次或重新设计所有权边界。
- 修复导致已验证测试退化：撤销该修复，不能修改旧测试迎合新实现。
- 缺少平台能力：记录真实 blocker 并失败退出，不能用桩函数假装完成。

详细分类和动作表见 `c-to-rust/references/error-recovery.md`。
