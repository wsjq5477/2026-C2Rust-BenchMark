# 渐进迁移规则

## 批次划分

批次是能够独立检查的最小依赖闭包，不等同于单个文件。推荐顺序为：公共常量与值类型、纯工具函数、I/O 抽象、共享数据库状态、业务状态机、公共 API、集成测试。

每批必须记录 C 输入符号、Rust 输出模块、直接依赖、测试、错误和验证命令。未经过测试的生成代码状态只能是 `generated`。

## Rust 设计默认值

- 使用 stable Rust 和标准库，避免评测时联网下载依赖。
- 错误通过枚举和 `Result` 传播，不吞掉 I/O 或数据损坏错误。
- 文件和锁由所有权对象管理，退出作用域自动释放。
- 地址、长度和对齐运算使用 checked arithmetic；越界必须返回错误。
- 持久化字段逐字段编码，明确字节序、宽度、状态位和擦除值。
- 缓存是优化层，关闭缓存时行为必须保持一致。

## 跨文件上下文

翻译函数前必须同时读取其类型定义、宏分支、全局状态、调用者、被调用者和覆盖它的测试。公共接口变更前先更新符号映射，并确认所有消费者在同一批次或已有适配。

## 完成定义

核心文件映射、公共符号映射和测试映射均为 `verified`；构建不再依赖 C 核心源码；质量门禁全部通过。
